http-rails Documentation

Welcome to the documentation for http-rails, a simple HTTP server framework for Node.js!

Installation

To install http-rails, use npm:

npm install http-rails

Getting Started

Step 1: Create a New JavaScript File

Create a new JavaScript file (e.g., app.js) in your project directory.

Step 2: Require Necessary Modules

Require the necessary modules in your JavaScript file:

//requires that are needed
    const path = require("path");
    const fs = require("fs");
    const rails = require("http-rails");

Usage

Step 3: Create an Instance of http-rails

Create an instance of http-rails:

const rail = new rails();

Step 4: Set the Port

Set the port number on which your server will listen:

const port = 3000;

Configuration

Step 5: Use Middleware

Use built-in middleware functions or create custom ones to handle requests: 

rail.use(rails.static(path.resolve("./public/")));

Step 6: Setup Error Pages

Set up custom error pages, such as a 404 page:

const missing = path.join(__dirname, "public", "404.html");
    rail.status("404", (req, res) => {
      res.sendFile(missing)
    });

Usage

Step 7: Define Routes

Define routes for your application:

rail.get("/joe", (req, res) => {
      res.setFileType("html");
      res.send("
joe
");
    });

Step 8: Spoof Routes

Create routes that point to existing files or paths:

rail.spoof("/cat", "/really-long-file-name-for-a-cat-file.html");

Usage

Step 9: Start the Server

Start the HTTP server and listen on the specified port:

rail.listen(port, () => {
      console.log(`Server is running on port ${port}`);
    });

Step 10: Run Your Application

Save your changes in the JavaScript file and run your application using Node.js: 

node app.js

Usage

Step 11: Access Your Website

Once your server is running, you can access your website by opening a web browser and navigating to http://localhost:3000 (or whichever port you specified).

Conclusion

Congratulations! You've now created a simple website using http-rails. You can further customize and expand your application by adding more routes, middleware, and functionality as needed.
